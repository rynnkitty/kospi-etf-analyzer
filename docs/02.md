# 📈 가치투자 종목 필터링 시스템 기획서

> **버전:** v1.1  
> **최종 수정일:** 2026-03-26  
> **목적:** PER·PBR 기반 저평가 종목 자동 발굴 시스템

---

## 1. 개요

본 프로젝트는 네이버 금융에서 수집한 시장 데이터(PER, PBR)를 분석하여, 기업의 내재 가치 대비 주가가 저평가된 종목을 체계적으로 발굴하는 것을 목적으로 한다. 감정이 아닌 데이터에 기반한 객관적 의사결정 체계를 구축하여, 수천 개 종목에서 투자 유망 후보 10~20개를 자동으로 압축한다.

## 2. 데이터 소스 및 수집

### 2.1 데이터 소스

| 항목 | 소스 | 비고 |
| :--- | :--- | :--- |
| PER, PBR, 배당수익률 | 네이버 금융 (finance.naver.com) | 종목별 스냅샷 |
| 업종 분류 | KRX 업종 분류 체계 | 한국거래소 기준 |
| 거래대금, 시가총액 | 네이버 금융 | 유동성·규모 판단용 |
| 재무제표 (ROE, 부채비율) | 네이버 금융 > 종목별 재무정보 | 가치 함정 방지용 |

### 2.2 수집 주기 및 방법

- **수집 주기:** 매 영업일 장 마감 후 1회 (일간 배치)

- **저장 형태:** CSV 또는 DB (종목코드, 종목명, 업종, PER, PBR, ROE, 부채비율, 배당수익률, 일평균거래대금, 시가총액)

## 3. 필터링 파이프라인

전체 필터링은 아래 **4단계 순차 파이프라인**으로 구성된다. 각 단계를 통과한 종목만 다음 단계로 진행한다.

```
[데이터 수집] → [1단계: 데이터 정제] → [2단계: 핵심 필터링] → [3단계: 그레이엄 필터] → [4단계: 스코어링 & 랭킹] → [결과 출력]
```

---

### 3.1 1단계: 데이터 정제 (Data Cleaning)

수치 오류 및 비정상적인 데이터를 제거하여 분석의 신뢰도를 높인다.

**제외 대상:**

| 제외 조건 | 사유 |
| :--- | :--- |
| PER ≤ 0 또는 PBR ≤ 0 | 당기순이익 적자 또는 자본 잠식 |
| 일평균 거래대금 1억 원 미만 (최근 20영업일) | 유동성 리스크 |
| 관리종목, 상장폐지 위험 종목 | 투자 부적격 |
| 스팩(SPAC) | PER/PBR 해석 불가 |
| 리츠(REITs) | 자산 구조 특수성 |
| 지주회사 | 연결 재무제표 왜곡 가능성 |

---

### 3.2 2단계: 핵심 필터링 (Core Filtering)

1단계를 통과한 종목에 대해 아래 **절대 기준**과 **상대 기준**을 **OR 조건**으로 적용한다. 둘 중 하나만 충족해도 통과시켜 업종 특성에 의한 과도한 필터링을 방지한다.

#### A. 절대 기준

| 지표 | 조건 | 비고 |
| :--- | :--- | :--- |
| PER | 0.5 < PER < 15 | 수익성 대비 저평가 여부 |
| PBR | 0.1 < PBR < 1.0 | 자산 가치 대비 저평가 (장부가치 이하) |

- PER과 PBR 조건은 **AND**로 적용 (두 조건 모두 충족 필요)

#### B. 상대 기준 (업종 내 비교)

| 지표 | 조건 | 비고 |
| :--- | :--- | :--- |
| PER | 해당 KRX 업종 평균 PER의 70% 이하 | 업종 특성 반영 |
| PBR | 해당 KRX 업종 평균 PBR의 70% 이하 | 업종 특성 반영 |

- PER과 PBR 조건 중 **하나 이상** 충족 시 통과

#### 조건 결합 방식 정리

```
통과 조건 = (절대 기준 A 충족) OR (상대 기준 B 충족)
```

> **설계 의도:** 절대 기준만 적용하면 PER이 원래 높은 테크 업종이 전부 탈락하고, 상대 기준만 적용하면 업종 전체가 고평가인 경우 비싼 종목도 통과할 수 있다. OR 조건으로 두 관점을 모두 허용하되, 이후 스코어링 단계에서 우열을 가린다.

---

### 3.3 3단계: 그레이엄 필터 (Graham Filter)

2단계를 통과한 종목에 **추가 적용**하는 보조 필터이다.

**조건:**

$$PER \times PBR \le 22.5$$

- 벤자민 그레이엄의 기준 (PER 15배 × PBR 1.5배 = 22.5)
- 두 지표 중 하나가 다소 높더라도 다른 하나가 충분히 낮으면 통과 가능
- **적용 방식:** 이 조건을 **충족하지 못하는 종목은 제외**한다.

---

### 3.4 4단계: 스코어링 & 랭킹 (Scoring & Ranking)

3단계까지 통과한 종목에 대해 **복합 점수(Composite Score)**를 산출하고 순위를 매긴다.

#### 스코어링 공식

$$Score = (w_1 \times S_{PER}) + (w_2 \times S_{PBR}) + (w_3 \times S_{ROE}) + (w_4 \times S_{SECTOR})$$

| 항목 | 산출 방식 | 설명 |
| :--- | :--- | :--- |
| $S_{PER}$ | 전체 후보 중 PER 낮은 순 백분위 순위 (0~1) | 낮을수록 고득점 |
| $S_{PBR}$ | 전체 후보 중 PBR 낮은 순 백분위 순위 (0~1) | 낮을수록 고득점 |
| $S_{ROE}$ | 전체 후보 중 ROE 높은 순 백분위 순위 (0~1) | 높을수록 고득점 |
| $S_{SECTOR}$ | 업종 내 상대 점수 (아래 참조) | 낮을수록 고득점 |

#### 업종 내 상대 점수 (Sector Score)

$$S_{SECTOR} = 1 - \left( \frac{기업\_PER}{업종\_평균\_PER} + \frac{기업\_PBR}{업종\_평균\_PBR} \right) \div 4$$

- 값이 1에 가까울수록 업종 내 상대적으로 더 저평가된 상태

#### 기본 가중치

| 가중치 | 기본값 | 조정 가이드 |
| :--- | :--- | :--- |
| $w_1$ (PER) | 0.30 | 수익성 중시 시 상향 |
| $w_2$ (PBR) | 0.25 | 자산가치 중시 시 상향 |
| $w_3$ (ROE) | 0.25 | 성장성 중시 시 상향 |
| $w_4$ (SECTOR) | 0.20 | 업종 비교 중시 시 상향 |

> **가중치 합계:** $w_1 + w_2 + w_3 + w_4 = 1.0$  
> 사용자가 투자 성향에 따라 조정할 수 있으며, 기본값은 균형형 포트폴리오 기준이다.

## 4. 가치 함정(Value Trap) 방지 필터

PER·PBR이 낮은 이유가 단순히 '싸서'가 아니라 '펀더멘탈이 나빠서'인 종목을 걸러내기 위한 추가 안전장치이다. 4단계 스코어링 **이전 또는 결과 출력 시 경고 표시용**으로 활용한다.

| 지표 | 기준 | 미달 시 처리 |
| :--- | :--- | :--- |
| ROE (자기자본이익률) | ≥ 5% | 결과에서 ⚠️ 경고 표시 |
| 부채비율 | ≤ 150% | 결과에서 ⚠️ 경고 표시 |
| 배당수익률 | ≥ 2% | 가점 부여 (Score +0.05) |

> **설계 의도:** ROE·부채비율 미달 종목을 즉시 제외하면 '턴어라운드' 가능 종목까지 놓칠 수 있으므로, 제외 대신 경고 표시 방식을 채택한다. 단, 사용자 설정에 따라 강제 제외 모드로 전환 가능하게 한다.

## 5. 결과 출력

### 5.1 출력 형태

| 형태 | 설명 |
| :--- | :--- |
| 콘솔/터미널 | 상위 20개 종목 테이블 출력 |
| CSV/Excel | 전체 필터링 결과 파일 저장 |
| 대시보드 (선택) | 웹 기반 시각화 (향후 고도화) |

### 5.2 출력 항목

```
순위 | 종목코드 | 종목명 | 업종(KRX) | PER | PBR | ROE | 부채비율 | 배당수익률 | 복합점수 | 경고
```

### 5.3 출력 예시

```
 1 | 005930 | 삼성전자   | 반도체    | 8.2 | 0.9 | 12.3% |  45% | 2.5% | 0.87 |
 2 | 055550 | 신한지주   | 은행      | 5.1 | 0.4 |  8.7% | 120% | 4.1% | 0.83 |
 3 | 009150 | 삼성전기   | 전자부품  | 7.5 | 0.7 |  6.2% |  60% | 1.8% | 0.79 | ⚠️ 배당
```

## 6. 백테스팅 설계

필터링 전략의 유효성을 과거 데이터로 검증한다.

### 6.1 백테스팅 조건

| 항목 | 설정 |
| :--- | :--- |
| 테스트 기간 | 최근 5년 (2021~2025) |
| 리밸런싱 주기 | 분기별 (3개월) |
| 포트폴리오 구성 | 필터링 상위 10~20종목 동일가중 |
| 벤치마크 | KOSPI 지수 수익률 |
| 수수료 | 매수·매도 각 0.015% (증권사 평균) |
| 슬리피지 | 0.1% (유동성 고려) |

### 6.2 평가 지표

- **CAGR (연평균 성장률):** 벤치마크 대비 초과 수익 여부
- **MDD (최대 낙폭):** 전략의 위험 수준
- **샤프 비율:** 위험 조정 수익률

## 9. 리스크 및 면책

> ⚠️ **투자 관련 면책 사항**  
> 본 시스템은 정량적 데이터 분석 도구이며, 투자 권유를 목적으로 하지 않는다. 필터링 결과는 참고 자료일 뿐이며, 최종 투자 판단은 사용자 본인의 책임 하에 이루어져야 한다. PER·PBR 등 과거 데이터 기반 분석은 미래 수익을 보장하지 않는다.
